### 2.3 技术经验问答

#### Swift语言

**问：** Can you tell me about your experience with Swift?
**答：** I've been using Swift for [X] years. I've developed several apps including [examples]. I particularly enjoy working with Swift's optionals and protocol-oriented features.

**问：** What is the difference between struct and class?
**答：** The main difference is that structs are value types and classes are reference types. Structs are copied when assigned, while classes share a single reference. Also, classes support inheritance, while structs don't.

**问：** How does ARC work in Swift?
**答：** ARC (Automatic Reference Counting) automatically manages memory by tracking strong references to objects. When the reference count drops to zero, the object is deallocated. We need to be careful with retain cycles by using weak or unowned references.

---

#### UI框架

**问：** Can you compare UIKit and SwiftUI?
**答：** UIKit is the older imperative framework, while SwiftUI is the newer declarative framework. UIKit offers more control and broader compatibility, while SwiftUI provides faster development and better state management. I use both depending on the project requirements.

**问：** How do you handle Auto Layout?
**答：** I use both Interface Builder and programmatic approaches with SnapKit. For complex layouts, I prefer SnapKit for better code readability and version control.

---

#### 架构设计

**问：** What architecture patterns have you used?
**答：** I've used MVC, MVVM, and VIPER. My preference is MVVM for most projects because it separates concerns well while keeping the code manageable. For larger projects, I've implemented Clean Architecture with dependency injection.

**问：** How do you handle dependency injection?
**答：** I use constructor injection most commonly. For view controllers, I inject dependencies through initializers. This makes testing much easier as I can mock dependencies.

---

### 2.4 行为问题 (STAR法则)

**STAR法则：Situation, Task, Action, Result**

**问：** Tell me about a challenging bug you fixed.
> **S:** I was working on an app that was crashing intermittently.
> **T:** My task was to identify and fix the root cause.
> **A:** I used Instruments to profile memory, added logging to track the crash, and discovered a retain cycle in the closure.
> **R:** The crash rate dropped from 2% to 0.1%, and the fix was documented for the team.

**问：** Describe a time you had a conflict with a teammate.
> **S:** My teammate and I disagreed on the architecture for a new feature.
> **T:** We needed to make a decision quickly to meet the deadline.
> **A:** I proposed we each create a prototype, then reviewed both approaches together. We chose elements from both.
> **R:** The feature was delivered on time, and we established a code review process.

**问：** Tell me about a time you missed a deadline.
> **S:** I underestimated the complexity of a payment integration feature.
> **T:** I was responsible for delivering the feature by end of sprint.
> **A:** I communicated early with the team, broke down the remaining work, and worked overtime to complete the critical path.
> **R:** We delivered a scaled-down version on time, with the full feature the following week.
