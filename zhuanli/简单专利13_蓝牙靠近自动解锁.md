# 专利13：一种蓝牙靠近自动解锁的省电方法

## 技术领域
本发明涉及iOS蓝牙交互技术领域，特别涉及一种基于蓝牙信号强度自适应检测距离的自动解锁方法。

## 背景技术
现有蓝牙解锁需要用户掏出手机触碰或点击，无法真正实现靠近自动解锁，且持续扫描蓝牙非常耗电。

## 发明内容

### 核心实现
```swift
class AdaptiveBLTUnlockManager: NSObject {

    private var centralManager: CBCentralManager!
    private var peripheral: CBPeripheral?
    private var lastRSSI: Int = 0

    // 动态阈值：基于历史数据自适应调整
    private var dynamicThreshold: Int = -70

    override init() {
        super.init()
        centralManager = CBCentralManager(delegate: self, queue: nil)
    }

    func startMonitoring() {
        centralManager.scanForPeripherals(withServices: [vehicleServiceUUID])
    }

    func adjustThreshold(basedOn history: [Int]) {
        // 根据历史信号强度调整阈值
        let avgRSSI = history.reduce(0, +) / history.count
        // 动态调整，保留安全余量
        dynamicThreshold = max(-80, avgRSSI - 10)
    }
}

extension AdaptiveBLTUnlockManager: CBCentralManagerDelegate {

    func centralManager(_ central: CBCentralManager,
                       didDiscover peripheral: CBPeripheral,
                       advertisementData: [String: Any],
                       rssi RSSI: NSNumber) {

        lastRSSI = RSSI.intValue

        // 记录历史用于阈值学习
        rssiHistory.append(RSSI.intValue)
        if rssiHistory.count > 20 {
            rssiHistory.removeFirst()
            adjustThreshold(basedOn: rssiHistory)
        }

        // 判断是否在有效距离内
        if RSSI.intValue > dynamicThreshold {
            // 靠近解锁
            triggerUnlock()
        }
    }

    private func triggerUnlock() {
        // 防抖：信号必须稳定超过阈值
        guard isSignalStable() else { return }
        VehicleSDK.shared.unlock()
    }

    private func isSignalStable() -> Bool {
        // 连续3次信号都超过阈值才触发
        return rssiHistory.suffix(3).allSatisfy { $0 > dynamicThreshold }
    }
}
```

## 优点
1. 靠近自动解锁，无需操作
2. 自适应阈值，误触发率低
3. 信号稳定性检测，防误触
