# 一种基于无感更新的微应用加载方法及系统

## 技术领域

本发明涉及移动应用技术领域，具体涉及一种微应用的更新方法和系统，特别涉及一种在使用中的微应用进行无感更新的方法和装置。

## 背景技术

随着移动互联网的发展，微应用（Mini App）作为一种轻量级应用形态被广泛采用。微应用通常需要从服务器下载完整的资源包（包括HTML、CSS、JavaScript等文件），然后在本地加载运行。

在现有的微应用更新方案中，存在以下问题：

1. **热更新冲突问题**：当用户正在使用某个微应用时，如果该微应用有新版本发布并开始下载更新，可能导致当前运行的微应用出现资源加载异常、功能紊乱甚至崩溃等问题。

2. **更新提示干扰**：现有方案通常在检测到新版本时弹出提示框要求用户确认或重启，这打断了用户的正常使用流程，影响用户体验。

3. **后台更新不可控**：即使采用后台下载更新，现有方案无法区分"首页微应用"和"普通微应用"，对所有微应用采用相同的更新策略，可能导致正在使用的首页微应用被更新覆盖。

因此，需要一种能够在用户使用微应用时静默下载新版本，并在下次启动时平滑切换的微应用更新方法。

## 发明内容

本发明提供一种基于无感更新的微应用加载方法及系统，旨在解决现有技术中微应用更新时打断用户使用、技术方案区分度低的问题。

### 发明目的

1. 实现微应用更新时不影响当前使用体验；
2. 对首页微应用采用特殊更新策略，确保用户无感知；
3. 提供一种可复用的微应用无感更新框架。

### 技术方案

#### 1. 核心策略：临时目录缓存 + 延迟替换

本发明的核心思想是：当检测到正在使用的微应用有新版本时，不直接覆盖当前版本，而是将新版本下载到一个独立的临时目录中，待用户下次启动该微应用时，再将临时目录中的新版本替换到正式目录，实现平滑过渡。

#### 2. 首页微应用识别机制

通过 `isBelongHomeMicroApp` 方法判断当前微应用是否为"首页微应用"：

```swift
class func isBelongHomeMicroApp(mappPackName: String) -> Bool {
    for homeMicroName in CAMicroDownLoadManager.autoDownloadMicroArray {
        if homeMicroName.count > 0,
           mappPackName.contains(homeMicroName) {
            return true
        }
    }
    return false
}
```

其中 `autoDownloadMicroArray` 预定义了首页微应用的包名白名单（如 `["micro.cn.com.uni.carModelControl"]`），只有匹配白名单的微应用才会触发无感更新策略。

#### 3. 双目录管理机制

系统维护两个目录：

- **正式目录** (`microDistPath`)：存储当前正在使用的微应用版本
- **临时目录** (`microDistPath/tmp`)：缓存新下载的微应用版本

```
微应用存储结构：
├── dist/                          # 微应用根目录
│   ├── micro.cn.com.uni.app1/     # 正式目录（当前使用版本）
│   ├── micro.cn.com.uni.app2/    # 正式目录
│   └── tmp/                       # 临时目录（新版本缓存）
│       └── micro.cn.com.uni.app1/ # 待替换的新版本
```

#### 4. 更新流程

**阶段一：后台静默下载**

```
1. 检测到新版本
2. 判断是否为首页微应用（isBelongHomeMicroApp）
3. 如果是首页微应用：
   - 下载到 tmp 临时目录
   - 不影响当前运行版本
4. 如果是普通微应用：
   - 直接下载到正式目录（可覆盖）
```

**阶段二：下次启动替换**

```
1. 用户打开微应用
2. 检查 tmp 目录是否存在对应新版本
3. 如果存在：
   - 将 tmp 目录内容移动到正式目录
   - 删除 tmp 目录中的缓存
   - 加载新版本
4. 如果不存在：
   - 直接加载正式目录版本
```

#### 5. 关键代码实现

**下载时判断首页微应用**

```swift
func unZipOpenFileWithZipPath(detailModel: CAMicroDetailModel, ...) {
    if CAMicroDownLoadManager.isBelongHomeMicroApp(mappPackName: detailModel.mappPackName) {
        /// home微应用解压到临时目录
        let tmpPath = CAMicroTools.microDistPath + "/tmp"

        /// 清空tmp之前下载的当前微应用
        CAMicroTools.removeFullPathDoc(fullPath: tmpPath + "/\(detailModel.mappPackName)")

        /// 解压到tmp文件夹
        if SSZipArchive.unzipFile(atPath: zipPath, toDestination: tmpPath, delegate: nil) {
            // 解压成功，等待下次启动时替换
        }
    } else {
        /// 普通微应用直接解压到主目录（覆盖式更新）
        if SSZipArchive.unzipFile(atPath: zipPath, toDestination: CAMicroTools.microDistPath, ...) {
            // 立即生效
        }
    }
}
```

**启动时检测并替换**

```swift
class func getMicroDocName(mappPackName: String) -> String {
    // ...
    /// 非主应用
    let changedName = getMicroPrefixName(mappPackName: mappPackName)

    // 首先检查是否有待替换的新版本在tmp目录
    if let docName = CAMicroDownLoadManager.maxHomeMicroDocumentDic[changedName],
       CAMicroTools.microExistDocLocalPath(docName: docName).count > 0 {
        return docName
    }

    /// 默认路径
    let mappPath = CAMicroTools.microDistPath.appendingPathComponent(changedName)
    if FileManager.default.fileExists(atPath: mappPath) {
        return changedName
    }
    return ""
}
```

#### 6. 版本号管理

系统通过 `buildId` 和 `versionId` 双重维度管理版本：

```swift
let localVersion = CAMicroTools.fetchLocalMicroVersion(docName: localMappPackName)
let currentVersion = detailModel.buildId
return CAMicroTools.compareVersions(v1: currentVersion, v2: localVersion)
```

只有在服务器版本号大于本地版本号时才触发更新。

#### 7. 防重复机制

通过 `goingMicroArray` 防止同一微应用被重复打开：

```swift
if goingMicroArray.contains(mappPackNameID) {
    return  // 已在打开中，防止重复
}
goingMicroArray.append(mappPackNameID)
// ... 使用完毕后移除
goingMicroArray.removeAll(where: { $0 == mappPackNameID })
```

## 具体实施方式

### 实施例一：首页微应用的无感更新

1. 用户打开并使用"车辆控制"首页微应用（`micro.cn.com.uni.carModelControl`）；

2. 系统检测到服务器有该微应用新版本（`versionId` 增加）；

3. 系统调用 `isBelongHomeMicroApp` 判断该微应用属于首页微应用；

4. 系统将新版本下载包解压至 `dist/tmp/micro.cn.com.uni.carModelControl/` 目录；

5. 用户继续使用当前版本，不受更新影响；

6. 用户下次打开该微应用时，系统检测到 tmp 目录有新版本，执行替换操作；

7. 替换完成后，用户看到的是最新版本。

### 实施例二：普通微应用的直接更新

1. 用户打开"设置"微应用；

2. 系统检测到服务器有新版本；

3. 系统调用 `isBelongHomeMicroApp` 判断该微应用不属于首页微应用；

4. 系统直接下载并解压到 `dist/micro.cn.com.uni.setting/` 目录，覆盖旧版本；

5. 用户下次打开时，直接加载新版本。

## 权利要求书

### 独立权利要求

**1. 一种基于无感更新的微应用加载方法，其特征在于，包括以下步骤：**

- **识别步骤**：当检测到微应用有新版本时，判断该微应用是否属于预设的首页微应用类型；
- **分类下载步骤**：若所述微应用属于所述首页微应用类型，则将新版本下载至临时存储目录；若所述微应用不属于所述首页微应用类型，则将新版本直接下载至正式存储目录并覆盖旧版本；
- **替换步骤**：当用户再次启动所述首页微应用时，将所述临时存储目录中的新版本内容移动至所述正式存储目录，完成版本替换。

**2. 一种基于无感更新的微应用加载系统，其特征在于，包括：**

- **首页识别模块**：用于判断微应用是否属于首页微应用类型；
- **分类下载模块**：用于根据识别结果将新版本下载至对应的存储目录；
- **延迟替换模块**：用于在用户下次启动首页微应用时执行版本替换。

### 对比现有技术的优势

| 特性 | 现有技术 | 本发明 |

| 更新时用户感知 | 弹出提示或中断 | 完全无感知 |
| 首页微应用区分 | 无区分，统一策略 | 特殊策略保护 |

## 摘要

本发明公开了一种基于无感更新的微应用加载方法及系统。该方法通过识别首页微应用类型，对正在使用的首页微应用采用临时目录缓存策略，将新版本下载到临时目录而非直接覆盖当前版本，待用户下次启动时再执行版本替换，实现了微应用更新过程中的用户无感知体验。该方案特别适用于首页类微应用的更新场景，有效避免了更新过程中对用户使用体验的影响，提升了整体系统的可用性和用户满意度。
